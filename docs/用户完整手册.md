## 1. 项目是做什么的？

NovelReader 是一套**小说阅读系统**，包含：

| 组成部分 | 路径 | 你能做什么 |
|---------|------|-----------|
| **API 服务端** | `apps/server` | 用户登录、书源、书架、上传、搜索等后台能力 |
| **网页端** | `apps/web` | 浏览器打开，功能最全，适合电脑日常使用 |
| **Windows 桌面端** | `apps/desktop` | 小悬浮窗，置顶、透明，边工作边读 |
| **Android 端** | `apps/mobile` | 手机 APK，书源/书架存在手机里 |
| **共享库** | `packages/*` | 书源引擎、TXT/EPUB 解析等（不用你直接操作） |

**重要：** 网页、桌面、手机都要能连上 **API 服务端** 才能搜索在线书源、登录同步。  
只有「纯本地打开 TXT」之类可以不完全依赖服务器（网页用浏览器临时打开）。

## 2. 各端区别与数据存在哪里

很多新手会搞混「浏览器」和「手机」存的数据，请看下表：

| 数据类型 | 网页端（游客） | 网页端（已登录） | Android APK | Windows 桌面 |
|---------|---------------|-----------------|-------------|-------------|
| 书源 | 浏览器 localStorage | 服务器数据库 | 手机 AsyncStorage | 用云端书源（需登录） |
| 书架 | 浏览器本地 | 服务器数据库 | 手机本地 + 可同步云端 | 服务器（需登录） |
| 阅读进度 | 浏览器本地 | 服务器数据库 | 手机本地 + 可同步云端 | 服务器（需登录） |
| 上传的 TXT/EPUB | — | 服务器磁盘 | 需登录走 API | 需登录走 API |
| 阅读主题设置 | 浏览器 / 云端 | 服务器数据库 | 手机本地 | 服务器（需登录） |

**一句话记忆：**

- **网页游客** = 数据在浏览器里，换浏览器或清缓存可能丢。
- **手机 APK** = 书源和本地书架在手机里，不是浏览器。
- **登录后** = 书架、进度、上传文件可云端同步（存在 PostgreSQL + 服务器 `uploads` 目录）。

## 3. 环境准备（第一次必做）

以下以 **Windows 10/11** 为例（你当前系统）。

### 3.1 安装 Node.js（必装）

1. 打开 [https://nodejs.org](https://nodejs.org)
2. 下载 **LTS 版本**（推荐 20 或 22）
3. 一路「下一步」安装
4. 打开 **PowerShell**，输入验证：

```powershell
node -v
```

应显示类似 `v22.x.x`。

### 3.2 安装 pnpm（必装）

在 PowerShell 中执行：

```powershell
npm install -g pnpm@9
pnpm -v
```

应显示 `9.x.x`。

### 3.3 安装 Docker Desktop（强烈推荐）

数据库（PostgreSQL）和 Redis 通过 Docker 一键启动，**小白强烈建议安装**。

1. 下载：[Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
2. 安装时若提示需要 **WSL 2**，按提示安装并重启
3. 若安装失败提示 **「Server 服务未启用」**：
   - 按 `Win + R`，输入 `services.msc` 回车
   - 找到 **Server**（或 LanmanServer）
   - 右键 → 属性 → 启动类型改为 **手动** → 点击 **启动** → 确定
   - 重新运行 Docker 安装程序
4. 安装完成后打开 Docker Desktop，等左下角显示 **Engine running**

验证：

```powershell
docker --version
docker compose version
```

### 3.4 Windows 桌面端额外依赖（仅桌面需要）

桌面端基于 **Tauri 2**，需要 **Rust**：

1. 打开 [https://rustup.rs](https://rustup.rs)
2. 下载并运行 `rustup-init.exe`，选默认安装
3. 安装 **Visual Studio C++ 构建工具**（若 `tauri dev` 报错再装）  
   [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) → 勾选「使用 C++ 的桌面开发」

### 3.5 Android 端额外依赖（仅手机开发需要）

| 工具 | 用途 | 安装 |
|------|------|------|
| Android Studio | 模拟器 / SDK | [developer.android.com/studio](https://developer.android.com/studio) |
| Expo Go（可选） | 真机快速调试 | 手机应用商店搜「Expo Go」 |

打包正式 APK 还需注册 [Expo 账号](https://expo.dev) 并安装 EAS CLI（见第七章）。

## 4. 服务端部署（核心，其他端都依赖它）

### 4.1 获取项目代码

若你已有文件夹 `novel-reader`，在 PowerShell 中进入：

```powershell
cd C:\Users\lenovo\Projects\novel-reader
```

（路径按你实际存放位置修改。）

### 4.2 配置环境变量

```powershell
copy .env.example .env
```

用记事本打开 `.env`，默认即可本地使用：

```env
DATABASE_URL="postgresql://novel:novel123@localhost:5432/novel_reader"
REDIS_URL="redis://localhost:6379"
JWT_SECRET="change-me-in-production-use-long-random-string"
PORT=3001
UPLOAD_DIR="./uploads"
CORS_ORIGIN="http://localhost:5173,http://localhost:1420"
```

> 上线公网时务必修改 `JWT_SECRET` 为长随机字符串。

### 4.3 一键启动（推荐小白）

```powershell
.\scripts\start.ps1
```

脚本会自动：

1. 创建 `.env`（若没有）
2. `docker compose up -d` 启动数据库
3. `pnpm install` 安装依赖
4. `pnpm db:push` + `pnpm db:seed` 初始化数据库
5. 打开两个新窗口分别跑 **API** 和 **网页**

看到以下地址即成功：

| 服务 | 地址 |
|------|------|
| 网页 | http://localhost:5173 |
| API 健康检查 | http://localhost:3001/api/health |

浏览器打开健康检查应返回 `{"status":"ok"}` 之类 JSON。

### 4.4 手动分步启动（想弄清每一步时用）

**终端 1 — 数据库：**

```powershell
cd C:\Users\lenovo\Projects\novel-reader

```

**终端 2 — 安装与初始化（只需第一次或依赖变更后）：**

```powershell

pnpm db:push
pnpm db:seed
```

**终端 3 — API：**

```powershell
pnpm --filter @novel-reader/server dev
```

**终端 4 — 网页：**

```powershell
pnpm --filter @novel-reader/web dev
```

### 4.5 预置演示账号

| 角色 | 邮箱 | 密码 |
|------|------|------|
| 普通用户 | `demo@novel.local` | `demo123` |
| 管理员 | `admin@novel.local` | `admin123` |

### 4.6 生产环境部署要点（进阶）

1. 服务器安装 Docker、Node 20+、pnpm
2. 修改 `.env` 中数据库密码、`JWT_SECRET`、`CORS_ORIGIN`（填你的域名）
3. `docker compose up -d`
4. `pnpm install && pnpm db:push && pnpm db:seed`
5. 构建并运行 API：`pnpm --filter @novel-reader/server build` → `node apps/server/dist/main.js`
6. 构建网页：`pnpm --filter @novel-reader/web build`，用 Nginx 托管 `apps/web/dist`
7. 手机/桌面连接时 API 地址改为 `https://你的域名/api`

## 5. 网页端：安装与使用

### 5.1 如何打开

服务端 + 网页 dev 启动后，浏览器访问：**http://localhost:5173**

### 5.2 游客模式（不登录）

登录页点击 **「游客继续使用（无需登录）」**。

游客可以：

- 书源商店 / 导入 JSON / 订阅链接（存在**本浏览器**）
- 搜索小说、在线阅读
- 本地打开 TXT（不上传服务器）
- 书源生成向导

游客不可以（需登录）：

- 云端上传 EPUB/TXT
- 书架云端同步、多设备进度同步
- AI / 词典工具（若服务端开启权限）

### 5.3 注册与登录

1. 点击「注册」→ 填邮箱、密码、昵称
2. 或使用演示账号 `demo@novel.local` / `demo123`

登录后侧边栏功能：

| 菜单 | 作用 |
|------|------|
| 书架 | 已收藏的书（含云端上传的） |
| 搜索 | 多书源搜索 |
| 书源商店 | 一键导入演示书源 |
| 我的书源 | 管理 Legado 书源 |
| 生成书源 | 输入网址半自动生成规则 |
| 上传书籍 | 云端上传或本地打开 |
| 设置 | 阅读主题、教程 |
| 管理后台 | 仅管理员可见 |

### 5.4 推荐阅读流程（小白）

```
书源商店 → 导入书源
    ↓
搜索 → 加入书架
    ↓
书架 → 点击阅读
```

### 5.5 上传 TXT / EPUB

- **上传到云端**：登录后，在「上传书籍」选文件，解析完自动进书架，换设备登录也能看
- **浏览器本地打开 TXT**：不登录也行，刷新页面需重新选文件
- **EPUB 本地打开**：网页端建议走「上传到云端」；本地直接打开 EPUB 能力有限

## 7. Android 手机客户端（APK）

### 7.1 存储说明（重要）

Android 端**不使用 SQLite**，当前使用：

- **AsyncStorage**：书源、书架、进度、API 地址
- **SecureStore**：登录 Token

数据在**手机应用私有目录**，与电脑浏览器无关。

### 7.2 开发调试

**1. 启动 API（电脑上）**

```powershell
pnpm --filter @novel-reader/server dev
```

**2. 查电脑局域网 IP**

```powershell
ipconfig
```

记下类似 `192.168.1.10` 的 IPv4 地址。

**3. 启动 Android 项目**

```powershell
pnpm --filter @novel-reader/mobile android
```

- **Android 模拟器**：APP 内默认 API 为 `http://10.0.2.2:3001/api`（10.0.2.2 代表宿主机）
- **真机**：手机与电脑同一 WiFi，在 APP「设置」填 `http://192.168.1.10:3001/api`

**4. 防火墙**

若手机连不上 API，在 Windows 防火墙中允许 **3001 端口** 入站，或临时关闭防火墙测试。

### 7.3 APP 使用说明

底部四个 Tab：

| Tab | 功能 |
|-----|------|
| 书架 | 本地书架 + 登录后云端书架 |
| 搜索 | 用本机书源或云端书源搜索 |
| 书源 | 粘贴 JSON、订阅链接、从商店导入 |
| 设置 | API 地址、登录/退出 |

**推荐流程（可不登录）：**

1. 书源 → 粘贴 Legado JSON 或从商店导入
2. 搜索 → 加入书架
3. 书架 → 阅读（进度自动存手机）

**登录后：** 可同步云端书架上的书；本机导入的书源仍在手机里。

### 7.4 打包 APK

```powershell
npm install -g eas-cli
cd apps/mobile
eas login
eas build --platform android --profile preview
```

`preview` 配置在 `apps/mobile/eas.json`，产出 **apk** 文件，下载到手机安装即可。

> 国内手机可能需开启「允许安装未知来源应用」。

## 8. 账号、游客模式与权限

### 8.1 三种身份

| 身份 | 说明 |
|------|------|
| 游客 | 不登录，权限由管理员「游客权限」控制 |

| 权限 | 含义 |
|------|------|

| 搜索书籍 | 能否搜索 |

## 9. 书源：导入、搜索、阅读（新手指南）

### 9.1 什么是书源？

书源是一套 **Legado 格式** 的规则 JSON，告诉程序如何从小说网站搜索、抓目录、抓正文。  
你不需要会写代码，**导入别人分享的书源**即可。

### 9.2 三种获取书源的方式

**方式 A：书源商店（最简单）**

网页/手机 → 书源商店 → 点「导入」

**方式 B：粘贴 JSON**

1. 在网上搜索「Legado 书源」找到 `.json` 或订阅链接
2. 我的书源 → 粘贴整段 JSON → 导入

**方式 C：书源生成向导**

1. 打开「生成书源」
2. 输入小说站首页 URL
3. 按步骤探测搜索页、目录页、正文页
4. 保存

### 9.3 书源 JSON 示例格式

```json
[
  {
    "bookSourceName": "示例书源",
    "bookSourceUrl": "https://example.com",
    "searchUrl": "...",
    "ruleSearch": { ... },
    "ruleToc": { ... },
    "ruleContent": { ... }
  }
]
```

### 9.4 搜索不到结果？

1. 检查书源是否 **已启用**
2. 换关键词（书名不要太长）
3. 书源可能失效，换别的书源
4. 确认 API 服务已启动、网络正常

## 11. 常见问题与排错

### Q1：`docker` 不是内部或外部命令

→ 未安装 Docker 或未加入 PATH。安装 Docker Desktop 并重启 PowerShell。

### Q2：Docker 安装失败 Server 服务

→ 见 [3.3 节](#33-安装-docker-desktop强烈推荐)，启用 Windows **Server** 服务后重装。

### Q3：`pnpm db:push` 连接数据库失败

→ 执行 `docker compose ps` 看 postgres 是否 Up。  
→ 检查 `.env` 里 `DATABASE_URL` 是否与 `docker-compose.yml` 一致（用户 `novel`，密码 `novel123`）。

### Q4：网页打开一片空白 / API 报错

→ 确认两个终端都在跑：`server dev` 和 `web dev`。  
→ 访问 http://localhost:3001/api/health 是否正常。

### Q5：手机连不上 API

→ 手机和电脑同一 WiFi。  
→ 设置里 API 用电脑 IP，不要用 `localhost`。  
→ 关闭电脑防火墙或放行 3001 端口。

### Q6：桌面 `tauri dev` 编译失败

→ 安装 Rust 和 MSVC 构建工具（3.4 节）。  
→ 关闭占用 1420 端口的程序。

### Q7：游客书源丢了

→ 网页：清了浏览器数据或换了浏览器。  
→ 手机：卸载 APP 会清数据。重要书源请 **登录后导入到云端** 或自行备份 JSON。

### Q8：上传 EPUB 一直「解析中」

→ 看 API 终端报错；文件是否超过 50MB；是否已登录且有「云端上传」权限。

### Q9：如何跑测试确认项目正常？

```powershell
cd C:\Users\lenovo\Projects\novel-reader
pnpm test
```
